HTTP/HTTPS

HTTP

HTTP stands for Hypertext Transfer Protocol. It is a protocol used for communication between web browsers and web servers. HTTP allows the transfer of hypertext, which includes HTML documents, images, videos, and other resources, over the internet.

When you type a URL (Uniform Resource Locator) into a web browser, such as Chrome or Firefox, the browser sends an HTTP request to the web server hosting that URL. The server then responds to the request, sending back the requested data, typically in the form of an HTML document.

HTTP follows a client-server model, where the client (typically a web browser) initiates a request to the server, and the server responds with the requested data or an error message. The communication between the client and the server happens over a TCP/IP connection.



HTTPS

HTTPS stands for Hypertext Transfer Protocol Secure. It is a protocol used for secure communication over a computer network, most commonly the internet. HTTPS is the secure version of HTTP, which is the protocol used for transferring data between a web browser and a website.

The primary purpose of HTTPS is to provide encryption and secure authentication of the website that users are connecting to. It ensures that the data exchanged between the browser and the website remains confidential and cannot be intercepted or tampered with by attackers.

HTTPS uses SSL (Secure Sockets Layer) or TLS (Transport Layer Security) protocols to encrypt the data. When a user connects to a website over HTTPS, their browser and the website's server establish a secure connection by exchanging SSL/TLS certificates. These certificates verify the authenticity of the website and enable the encryption of data transmitted between the two parties.

Domain

In the context of computer networking and the Internet, a domain refers to a unique name that identifies a website or a network of devices. It is part of a larger hierarchical naming system called the Domain Name System (DNS), which translates domain names into their corresponding IP addresses.

A domain name is typically composed of two or more parts, separated by dots. The rightmost part is the top-level domain (TLD), which represents the highest level in the domain name hierarchy. Examples of TLDs include .com, .org, .net, .edu, .gov, and country-specific TLDs such as .uk, .de, or .jp.

The part to the left of the TLD is the second-level domain (SLD) and is often chosen to represent a specific organization, business, or topic. For example, in the domain name "example.com," "example" is the SLD. Further subdomains can be added to the left of the SLD to create additional levels of organization or categorization. For instance, "www" is a commonly used subdomain for the World Wide Web, resulting in the complete domain name "www.example.com."



Hosting

Hosting refers to the process of storing, serving, and making websites or web applications accessible on the Internet. It involves allocating resources, such as disk space, bandwidth, and computing power, to host and serve the files and data associated with a website.

Web hosting providers are companies that offer services and technologies to store and serve websites on their servers. When you sign up for a hosting service, you essentially rent space on their servers to store your website files, databases, and other related assets. The hosting provider ensures that your website is available to be accessed by users on the Internet.

There are different types of hosting services available, including:

1. Shared Hosting: In shared hosting, multiple websites are hosted on a single server, and resources such as CPU, memory, and disk space are shared among them. This is a cost-effective option suitable for small websites or those with moderate traffic.

2. Virtual Private Server (VPS) Hosting: VPS hosting involves dividing a physical server into multiple virtual servers, where each website or application has its dedicated resources. It provides more control, privacy, and scalability compared to shared hosting.

3. Dedicated Server Hosting: With dedicated hosting, you have an entire physical server dedicated solely to your website or application. This option offers the highest level of control, performance, and flexibility, making it suitable for high-traffic websites or resource-intensive applications.

4. Cloud Hosting: Cloud hosting utilizes multiple interconnected servers to host websites or applications. It offers scalability and redundancy, as resources can be dynamically allocated based on demand. Cloud hosting can be a good option for websites with fluctuating traffic or for those requiring high availability.

5. Managed WordPress Hosting: This type of hosting is specifically optimized for WordPress websites. The hosting provider takes care of WordPress-specific tasks, such as updates, security, and performance optimizations, allowing you to focus on your website content.



Host

In web development, the term "host" refers to the act of making a website or web application accessible on the Internet by storing its files and data on a server that is connected to the World Wide Web. When you develop a website or web application, it typically consists of various files such as HTML, CSS, JavaScript, images, and databases. These files need to be stored on a server and made available for users to access through their web browsers.

To host a website or web application, you need to follow these steps:

1. Choose a hosting provider: Select a hosting provider that offers the type of hosting service that suits your website's requirements. Consider factors such as server reliability, storage space, bandwidth, technical support, and cost.

2. Register a domain: If you don't already have a domain name, you'll need to register one. A domain name is the unique address that users will use to access your website (e.g., www.example.com).

3. Set up the hosting account: Sign up for a hosting plan with the chosen provider and follow their instructions to set up your hosting account. This may involve creating an account, choosing a hosting package, and providing your domain name.

4. Upload website files: Use an FTP (File Transfer Protocol) client or the hosting provider's provided tools to upload your website files to the server. This includes all HTML, CSS, JavaScript, image, and other necessary files.

5. Configure databases and server settings: If your website requires a database (e.g., for storing user information or dynamic content), you will need to set up the database on the server and configure the necessary settings.

6. Test and launch: Once your website files are uploaded and configured, test your website to ensure that it functions correctly. Make sure all links work, forms submit data properly, and any dynamic functionality functions as expected. Once you are satisfied with the testing, you can officially launch your website, making it accessible to users on the Internet.



FTP

FTP, which stands for File Transfer Protocol, is a standard network protocol used for transferring files between a client and a server on a computer network. It provides a simple and efficient way to upload, download, and manage files on a remote server.

Here's how FTP typically works:

1. Client-Server Communication: The FTP client (usually a software application) establishes a connection with the FTP server using a specified port (usually port 21 for standard FTP). The client and server communicate over this connection using FTP commands and responses.

2. Authentication: The client provides credentials (username and password) to authenticate itself with the FTP server. Once authenticated, the client gains access to the files and directories available on the server.

3. Navigating the Directory Structure: The client can navigate the directory structure of the server to locate and access specific files or directories. FTP provides commands like "cd" (change directory), "ls" (list files and directories), and "pwd" (print working directory) to navigate and manage the server's file system.

4. File Transfer: The client can initiate file transfers between the local machine and the server. FTP supports commands such as "get" (download a file from the server to the client), "put" (upload a file from the client to the server), and "delete" (remove a file from the server). During file transfer, data is transmitted over a separate data connection established by the FTP protocol.

5. Passive and Active Modes: FTP can operate in two modes: passive and active. In the active mode, the server initiates the data connection back to the client for file transfers. In the passive mode, the client initiates the data connection to the server. The choice of mode depends on the network configuration and firewall settings.



Path

In web development, a "path" typically refers to the specific location or route within a website or web application. It is part of the URL (Uniform Resource Locator) that specifies the location of a particular resource or page on a server.

A path is represented as a string of characters that comes after the domain name in a URL. It helps the server identify the specific resource or page that the user is requesting. For example, in the URL "https://www.example.com/products/category1/item1" , the path "/products/category1/item1" represents the specific path to the item1 page within the category1 section of the products on the example.com website.



Extension

A filename extension, file name extension or file extension is a suffix to the name of a computer file (e.g., .txt, .docx, .md). The extension indicates a characteristic of the file contents or its intended use. A filename extension is typically delimited from the rest of the filename with a full stop (period), but in some systems it is separated with spaces. Other extension formats include dashes and/or underscores on early versions of Linux and some versions of IBM AIX.

Some file systems implement filename extensions as a feature of the file system itself and may limit the length and format of the extension, while others treat filename extensions as part of the filename without special distinction.

URL-URI/URN

URL

Defination of URL

A URL (Uniform Resource Locator) is a unique identifier used to locate a resource on the Internet. It is also referred to as a web address. URLs consist of multiple parts -- including a protocol and domain name -- that tell a web browser how and where to retrieve a resource.

URL -- Uniform Resource Locator

Contains information about how to fetch a resource from its location. For example:

http://example.com/mypage.html

ftp://example.com/download.zip

mailto:user@example.com

file:///home/user/file.txt

http://example.com/resource?foo=bar#fragment

/other/link.html (A relative URL, only useful in the context of another URL)

URLs always start with a protocol (http) and usually contain information such as the network host name (example.com) and often a document path (/foo/mypage.html). URLs may have query parameters and fragment identifiers.


URN -- Uniform Resource Name

Identifies a resource by name. It always starts with the prefix urn: For example:

urn:isbn:0451450523 to identify a book by its ISBN number.

urn:uuid:6e8bc430-9c3a-11d9-9669-0800200c9a66 a globally unique identifier

urn:publishing:book - An XML namespace that identifies the document as a type of book.

URNs can identify ideas and concepts. They are not restricted to identifying documents. When a URN does represent a document, it can be translated into a URL by a "resolver". The document can then be downloaded from the URL.

URI -- Uniform Resource Identifier

URIs encompasses both URLs, URNs, and other ways to indicate a resource.

An example of a URI that is neither a URL nor a URN would be a data URI such as data:,Hello%20World. It is not a URL or URN because the URI contains the data. It neither names it, nor tells you how to locate it over the network.

There are also uniform resource citations (URCs) that point to meta data about a document rather than to the document itself. An example of a URC would be an indicator for viewing the source code of a web page: view-source:http://example.com/ . A URC is another type of URI that is neither URL nor URN.



Deferance between URL, URI and URN

URI (uniform resource identifier) identifies a resource (text document, image file, etc) URL (uniform resource locator) is a subset of the URIs that include a network location. URN (uniform resource name) is a subset of URIs that include a name within a given space, but no location.



Address bar

The address bar is the familiar text field at the top of a web browser's graphical user interface (GUI) that displays the name or the URL (uniform resource locator) of the current web page. Users request websites and pages by typing either the name or the URL into the address bar.

The term address bar refers to the text field in a web browser that identifies the user's location on the web and allows them to access different websites. The address bar is known as a location bar, and in Google Chrome, it's called the omnibox.



Title bar

The title bar at the top of a window displays an application-defined icon and line of text. The text specifies the name of the application and indicates the purpose of the window. The title bar also makes it possible for the user to move the window using a mouse or other pointing device.
